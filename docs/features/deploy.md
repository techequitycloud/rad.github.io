# Deploy (Marketplace)

The **Deploy** page is the central marketplace where you can discover, explore, and deploy modules. It serves as the catalog of available infrastructure-as-code solutions.

## Features

### 1. Module Discovery
*   **Platform Modules:** Curated modules provided by the platform administrators. These are generally trusted and verified.
*   **Partner Modules:** Modules published by certified partners. You can switch between the "Platform" and "Partner" tabs to browse different catalogs.
*   **Search:** Use the search bar to find modules by name, keyword, or description.
*   **Pagination:** Browse through large collections of modules using the pagination controls at the bottom of the page.

### 2. Module Cards
Each module is represented by a card displaying:
*   **Name:** The name of the module.
*   **Description:** A brief overview of what the module does.
*   **Rating:** A star rating based on user feedback.
*   **Deployment Count:** The number of times this module has been deployed, indicating its popularity.
*   **Cost:** The estimated credit cost to deploy or run this module (if applicable).
*   **Actions:**
    *   **Deploy:** Click the card (or the deploy button) to start the deployment wizard.
    *   **Pin:** Click the "Pin" icon to save the module to your dashboard for quick access.
    *   **Delete:** Enables module owners to delete a module from the dashboard.

### 3. Dashboard Stats
At the top of the page, you'll see a summary of your account's activity:
*   **Deployment Counts:** Total number of deployments you have made.
*   **Credit Balance:** Your current available credits.
*   **Retention Period:** How long your deployment history is kept before being archived or deleted.

## How to Deploy a Module

1.  **Find a Module:** Browse the Platform or Partner tabs, or use the search bar.
2.  **Select:** Click on the module card you wish to deploy.
3.  **Configure:** You will be taken to a configuration screen (Provisioning) where you can input the necessary variables (e.g., region, instance size, names).
4.  **Review & Deploy:** Confirm your settings and click "Deploy".
5.  **Monitor:** You will be redirected to the **Deployments** page to watch the progress of your new deployment.
