Platform Capabilities
The Five Planes of the RAD Internal Developer Platform
RAD implements all five planes of the IDP model defined by the CNCF Platform Working Group and referenced in Gartner’s platform engineering framework
1 — Developer Control Plane

Self-Service Module Catalogue, Deployment UI & Credit-Based Billing

Teams deploy production infrastructure via a UI without engineering support, in minutes. Built-in subscription management (Stripe & Flutterwave), credit system, partner revenue sharing, and referral programme. Admin, User, Partner, Finance, Support, and Agent roles model a complete Cloud Centre of Excellence.

2 — Integration & Delivery Plane

Cloud Build, Artifact Registry & Cloud Deploy

Automated CI/CD with immutable image tagging, Kaniko builds, progressive deployment strategies, and secret management integration. Every module follows an Infrastructure-as-Code paradigm — reproducible, auditable, version-controlled. Jules AI (GitHub-integrated) provides continuous module refinement.

3 — Observability Plane

Cloud Monitoring, Cloud Logging, SLO/SLI Dashboards & Distributed Tracing

SRE-grade observability out of the box: four golden signals dashboards, multi-burn-rate alert policies, uptime checks, log-based metrics, and Cloud Trace integration. Every deployment is observable from day one.

4 — Security & Identity Plane

IAP (BeyondCorp), Cloud Armor, Secret Manager & Workload Identity

Zero-trust access on every deployment. No public IPs by default. Cloud NAT for private egress. VPC Service Controls for data perimeter. CMEK support. Automated secret rotation. Cloud Armor WAF with OWASP Top 10 rule sets. Binary authorisation for container trust. Every deployment is BeyondCorp-compliant.

5 — Resource Plane

GKE Autopilot, Cloud Run Gen 2, Cloud SQL, Cloud Filestore & Memorystore

Serverless-first architecture with managed auto-scaling. Multi-region deployment support. HA Cloud SQL with automated daily backups and point-in-time recovery. Redis caching. Scale-to-zero for cost optimisation. All configurable via the module UI — no Terraform knowledge required.
