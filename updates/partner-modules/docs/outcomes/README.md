# Outcomes

This folder documents the **business outcomes** delivered by the platform — what customers and partners gain, quantified where possible. Outcomes documents are curated indexes that frame value; the implementation mechanics live in their canonical homes under `docs/practices/` and `docs/capabilities/`.

## How outcomes, practices, and capabilities relate

| Layer | Folder | Answers |
|---|---|---|
| **Outcomes** | `docs/outcomes/` | *What does this deliver for my organisation?* |
| **Practices** | `docs/practices/` | *How do we operate it?* |
| **Capabilities** | `docs/capabilities/` | *How does it technically work?* |

## Cross-folder navigation

- Start with an outcome that matches your business question above.
- Follow cross-references into `docs/practices/` for operating guidance or `docs/capabilities/` for implementation detail.
- For certification exam preparation, start with `education_enablement.md` and follow the links to `docs/certification/`.
