The Root Cause

Decades-old architectural decisions continue to constrain transformation initiatives. Documentation is lost through staff turnover. Critical business logic is trapped in code that few understand and even fewer can modify safely.

As a result, organisations develop a culture of fear around touching “working” systems — where “working” often means “we’re afraid to verify whether it actually works correctly.” This paralysis manifests in risk-averse behaviour that becomes increasingly untenable as competitive pressure mounts.

The good news: you don’t have to choose between modernization and shipping your next feature.

Google’s Cloud Application Modernization Program
CAMP is Google's framework for designing a tailored, end-to-end application modernization journey — providing step-by-step guidance while allowing each organisation to choose their path

One size does not fit all. CAMP was designed with that in mind.

A company new to containerisation can start by understanding the benefits before moving to more complex patterns like multicloud or DevOps. CAMP provides the assessments, workshops and Pilots to meet organisations exactly where they are — and move them forward at a pace they can sustain.

  • DevOps Assessment — DORA-based analysis of software delivery practices; identifies highest-impact improvement areas across the organisation

Claude Code — The Architect’s Partner

Whole-Codebase Awareness

Claude Code understands context across entire codebases — tracing execution paths through multiple files, identifying dependencies before they become problems, and walking teams through the reasoning behind architectural decisions.

  • Analyse legacy codebases: extract business logic, map dependencies, document hidden constraints
  • Design migration architectures: monolith decomposition, service boundary identification
  • Generate test suites before any code is migrated — validate equivalence
  • Produce complete documentation with a single prompt
  • Enables junior developers to deliver architect-quality outputs
Jules — The Always-On Executor

Async Execution, GitHub-Native

Jules works asynchronously in a secure cloud VM — powered by Gemini 2.5 Pro — cloning repos, making changes, running tests, and opening pull requests while your team focuses elsewhere.

  • GitHub-native: understands full codebase context, works through issues, branches and PRs
  • Jules API embeds directly into RAD’s Cloud Build / Cloud Deploy pipelines
  • Jules Tools CLI: scriptable, composable with existing developer toolchains
  • AGENTS.md: encodes platform conventions Jules follows consistently
  • Multi-VM parallel execution: concurrent tasks across the entire codebase

“Claude Code for thinking. Jules for doing. RAD for deploying. CAMP for navigating.”

Together, these four layers compress months of platform engineering into weeks — and transform a one-time CAMP engagement into a continuously improving platform. Claude Code handles design; Jules handles execution; RAD handles infrastructure; CAMP handles the methodology that ties it all together.

Maturity Model
Platform Engineering Maturity
Where is your organisation on the platform engineering journey? Understanding your current maturity level is the starting point for every CAMP engagement — and the baseline against which improvement is measured
Enterprise + Partner
Level 1 — Ad Hoc Reactive & Undocumented

Legacy systems addressed only during failures, retiring programmers or regulatory deadlines. No documentation, test suites or transition plans. Knowledge lives solely in retiring experts’ heads. Technical debt compounds as teams add workarounds rather than touching core systems.

Level 2 — Planned Annual Projects, Siloed Efforts

Annual projects target specific systems, though selection reflects politics over business value. Each project reinvents the wheel. Basic legacy inventories exist without sophisticated ROI analysis. Some automated conversion tools help but humans still verify most translations manually.

Level 3 — Systematic Dedicated Teams, Standard Processes

Dedicated teams follow standardised processes with clear metrics. Living inventories track technical debt scores and modernization readiness. AI tools continuously analyse legacy code. Established playbooks guide common patterns. Automated testing ensures compatibility.

Level 4 — Optimised AI-Driven Continuous Improvement

AI agents proactively scan systems and generate modernization proposals. Jules reads legacy code and automatically creates cloud-native services with full test coverage. Continuous background modernization removes dead code, optimises queries and generates compliance updates. Humans focus on strategy.
