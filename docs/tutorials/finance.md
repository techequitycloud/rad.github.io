# Tutorial: Finance Dashboard

## Overview

The Finance role provides visibility into the platform's full economic picture — subscription tiers, credit pricing, individual user balances, agent commission revenue, and project-level invoicing. This tutorial walks through each section of the Billing dashboard and explains how to use it for ongoing financial management and compliance reporting.

**Audience:** Users with the **Finance** role  
**Estimated time:** 15–20 minutes

By the end of this tutorial you will have:
- Navigated all tabs of the Billing dashboard
- Reviewed agent commission revenue with date filtering
- Adjusted global credit settings and individual user balances
- Generated a project invoice for compliance or reporting purposes

---

## Step 1: Access the Billing Dashboard

1. Click **Billing** in the navigation bar.

   > **Note:** The Billing link is only visible to users who have been assigned the **Finance** role. If you do not see it, ask your platform Administrator to assign this role to your account.

2. When the credit and subscription features are enabled, the Billing page has five tabs:

   | Tab | Purpose |
   | :--- | :--- |
   | **Subscription Tiers** | Define and manage the pricing plans available to users |
   | **Credit Settings** | Set global credit pricing, sign-up bonuses, and monthly allocations |
   | **Credit Management** | Adjust credit balances for individual user accounts |
   | **Agent Revenue** | Review commission earnings generated by referring agents |
   | **Project Invoices** | Generate and download monthly invoices per project |

---

## Step 2: Review Subscription Tiers

Subscription tiers control what pricing plans users see when they visit the Credits page.

1. Click the **Subscription Tiers** tab.
2. The table lists all active plans, showing the tier name, price, credit allocation, and included features.
3. To add a new tier, click **Add New Tier** and complete the form. To edit an existing tier, click its row.
4. Changes take effect immediately — users will see updated plans the next time they visit the Buy Credits page.

---

## Step 3: Manage Credit Settings

Credit settings define the platform-wide economics of the credit system. Changes here affect all users.

1. Click the **Credit Settings** tab.
2. Review and update the following settings as needed, clicking **Save** after each change:

   | Setting | Description |
   | :--- | :--- |
   | **Price Per Credit** | The number of credits equivalent to one unit of currency (for example, `100` credits = £1) |
   | **Signup Credits** | Free credits awarded automatically to every new user on first login |
   | **Credits Per Hour** | The build-time charge rate — credits deducted per hour of deployment build time |
   | **Low Credit Notification** | The balance threshold at which the platform emails a low-credit warning to the user |
   | **Monthly Credits** | Recurring credits automatically added to every user's balance each month |

> **Note:** Changes to **Price Per Credit** affect future credit purchases only. Existing purchased credit balances are not recalculated.

---

## Step 4: Analyse Agent Revenue

The Agent Revenue tab shows commissions earned by users participating in the referral programme.

1. Click the **Agent Revenue** tab.
2. The dashboard shows a breakdown of:
   - Total revenue generated by referred users' deployments
   - Commission amounts owed to each agent
   - Paid vs. pending commission status
3. Use the **date range filter** to narrow the view to a specific reporting period (for example, the current calendar month or financial quarter).
4. Use this data to reconcile commission payments and verify that agent payouts match the figures in your accounts system.

---

## Step 5: Adjust Individual User Credits

When a user needs a manual credit correction — for example, after a billing dispute, a failed deployment that was not their fault, or an onboarding incentive — use Credit Management.

1. Click the **Credit Management** tab.
2. Use the search bar to find the user by email address.
3. Click **Edit** on their row.
4. Update the **Awards** field to the correct total. This field represents the cumulative awarded credits for that user — enter the new total, not the delta.
5. Click **Save**. The change is applied instantly.

---

## Step 6: Generate Project Invoices

Project invoices provide a formal record of deployment costs, suitable for compliance, client billing, or internal chargebacks.

1. Click the **Project Invoices** tab.
2. Select the **project** and **month** you want to generate an invoice for.
3. Click **Generate Invoice**. The platform calculates the total credit usage for that project and period.
4. Download the invoice in the available format (PDF or CSV) for distribution or archiving.

> **Tip:** Run invoice generation at the end of each calendar month as part of your month-end close process. Invoices reflect all deployments completed within the selected period, including build-time charges.
