# Screenshot Capture Guide for RAD Platform Tutorials

This document provides a comprehensive guide for capturing screenshots to complete the RAD Platform tutorials. Each screenshot is marked with `[SCREENSHOT: description]` in the tutorial files and needs to be replaced with actual images from the RAD Console.

## Screenshot Requirements

### General Guidelines

All screenshots should follow these standards:

- **Resolution**: Capture at 1920x1080 or higher resolution
- **Format**: Save as PNG for best quality
- **Browser**: Use Chrome or Firefox with a clean profile (no extensions visible)
- **Annotations**: Add numbered callouts, highlights, or arrows as needed
- **Consistency**: Use the same browser window size for all screenshots
- **Privacy**: Remove or blur any sensitive information (emails, project IDs, etc.)

### Annotation Standards

When annotating screenshots, use:

- **Red boxes or arrows** for primary focus areas
- **Numbered circles** (1, 2, 3) for step-by-step sequences
- **Yellow highlights** for important text or buttons
- **Blue boxes** for informational elements

## Screenshot Capture Workflow

1. **Prepare the environment**:
   - Use a clean browser profile
   - Set browser window to consistent size (1920x1080 recommended)
   - Clear any notifications or popups

2. **Capture screenshots**:
   - Use a screenshot tool (Snagit, Greenshot, or built-in OS tools)
   - Capture the entire browser window or just the relevant content area
   - Save with the specified filename

3. **Annotate screenshots**:
   - Use an image editing tool (Snagit, Photoshop, GIMP, or online tools)
   - Add arrows, boxes, numbers, and highlights as specified
   - Keep annotations clear and professional

4. **Save and organize**:
   - Save annotated screenshots as PNG files
   - Place in the correct directory: `/home/ubuntu/rad.github.io/static/img/tutorials/[role]/`
   - Use the exact filenames specified in this guide

5. **Update tutorial files**:
   - Replace `[SCREENSHOT: description]` markers with actual image references
   - Use Markdown syntax: `![Alt text](/img/tutorials/[role]/filename.png)`

## Example Markdown Image Syntax

```markdown
![RAD Console login page showing the Sign in with Google button](/img/tutorials/users/login-page.png)
```

## Tools Recommendations

### Screenshot Capture
- **Windows**: Snagit, Greenshot, or Windows Snipping Tool
- **macOS**: Snagit, CleanShot X, or built-in Screenshot app (Cmd+Shift+4)
- **Linux**: Flameshot, Shutter, or GNOME Screenshot

### Image Annotation
- **Snagit** (Windows/macOS) - Best all-in-one tool
- **GIMP** (Cross-platform) - Free and powerful
- **Photoshop** (Windows/macOS) - Professional option
- **Skitch** (macOS) - Simple and quick
- **Annotate.io** (Web-based) - No installation needed
